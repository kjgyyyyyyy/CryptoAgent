# CryptoAgent (ETH + BMNR)

개인용 크립토 리서치 에이전트 초안입니다.  
목표는 **ETH(이더리움) / BMNR(비트마인)** 중심으로 다음을 자동화하는 것입니다.

- 최신 뉴스/공시/리포트 수집
- 가격/수급/온체인/내러티브 분석
- 중요도 분류 + 요약/해석
- 질문응답(대화형) 지원
- 최종 리포트 생성

---

## 1) 에이전트 전체 흐름

1. 사용자 요청 받기
2. 가격 데이터 수집
3. 뉴스/공시/리포트 수집
4. 중요도 분류
5. 요약 + 해석
6. 최종 리포트 생성

---

## 2) 모듈 설계 (MVP)

### A. Intake 모듈 (요청 해석)
- 입력 예시
  - "ETH 오늘 급등 원인 정리해줘"
  - "BMNR 최근 7일 기관수급/내러티브 알려줘"
- 수행
  - 자산(ETH/BMNR), 기간(1D/7D/30D), 관점(트레이딩/중장기/온체인) 파싱
  - 리포트 타입 선택(브리프/딥다이브)

### B. Market Data 모듈 (가격/거래)
- 수집 항목
  - OHLCV, 거래대금, 변동성(ATR/표준편차), 펀딩비/미결제약정(가능 시)
- 산출 지표
  - 수익률(1D/7D/30D), 거래량 스파이크, 추세(EMA), 모멘텀(RSI)

### C. News & Docs 모듈 (뉴스/공시/리포트)
- 수집 항목
  - 거래소/프로토콜 공지, 기업 공시, 리서치 아티클, SNS 트렌드
- 메타데이터
  - 출처 신뢰도, 발행시각, 조회/공유 지표(관심도 proxy), 중복 여부

### D. Flows 모듈 (기관/ETF 수급)
- 기관수급
  - 대형 지갑 이동, 거래소 유입/유출, 장외 관련 단서
- ETF 수급
  - 순유입/순유출, 누적 AUM 변화, 전일 대비 변화율

### E. On-chain 모듈
- ETH 중심 지표 예시
  - 활성 주소, 신규 주소, 거래 건수, 가스비, 스테이킹 순증감, 거래소 보유량
- 해석 프레임
  - 가격 상승 + 거래소 유출 증가 => 매도 압력 완화 가능성
  - 가격 상승 + 거래소 유입 증가 => 차익실현 가능성

### F. Narrative 모듈
- 내러티브 태그
  - ETF, L2, AI+Crypto, RWA, 규제, 거시(금리/달러)
- 점수화
  - 언급량, 가속도(증감률), 출처 신뢰도 가중치로 내러티브 점수 계산

### G. Ranking 모듈 (중요도 분류)
- 중요도 점수 예시
  - `impact(0~5) + credibility(0~5) + market_relevance(0~5) + novelty(0~5)`
- 등급
  - S(매우 중요), A(중요), B(참고), C(노이즈)

### H. Synthesizer 모듈 (요약 + 해석)
- 출력 규칙
  - 팩트(관측)와 해석(의견) 분리
  - 상승/하락 시나리오 각각 제시
  - 신뢰도/한계(데이터 지연, 미확인 루머) 명시

### I. Report 모듈 (최종 리포트)
- 템플릿
  1) Executive Summary (3~5줄)
  2) 핵심 차트/수급/온체인 요약
  3) Top Articles/이슈 (중요도 순)
  4) Bull / Base / Bear 시나리오
  5) 오늘의 액션 포인트(관찰 포인트)

## 3) ETH / BMNR 집중 관찰 항목

### ETH
- 현물/파생 지표 동조 여부(가격 vs OI vs 펀딩비)
- ETF 순유입과 가격 탄력성
- 스테이킹/언스테이킹 추세, L2 활동 전이

### BMNR
- 기업 관련 공시/실적/사업 확장 뉴스
- 거래량 급증 구간의 이벤트 동반 여부
- 업종/테마(채굴, 반도체, 전력비용, BTC 민감도) 연결 분석

## 4) 질문응답(QA) 동작 방식

- 사용자 질문을 **의도 분류**
  - 사실 조회형 / 비교형 / 원인분석형 / 전망형
- 답변 규칙
  - 근거 3개 이내 핵심 bullet
  - 필요한 경우 즉시 후속 질문(기간, 기준통화, 리스크 선호)
- 예시
  - Q: "ETH 지금 매수 타이밍 어때?"
  - A: "단정 대신 시나리오로 답변" + 핵심 데이터 기준 제시

## 5) 초기 운영 정책

- 데이터 결측 시 "확인 불가"를 명확히 표기
